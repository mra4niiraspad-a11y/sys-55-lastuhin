Задание 1

1. Какие сетевые службы в ней разрешены?

По результатам сканирования nmap на целевой машине (IP 10.0.2.15) обнаружены следующие открытые сетевые службы:

rpcbind (порт 111/tcp, версия 2–4, RPC #100000) — служба регистрации и поиска RPC‑сервисов (также известна как portmapper).

nfs_acl (порт 2049/tcp, версия 3, RPC #100227) — служба NFS (Network File System) с поддержкой списков контроля доступа (ACL), позволяющая монтировать файловые системы по сети.

2. Какие уязвимости были вами обнаружены?

На основе найденных служб выявлены следующие уязвимости (достаточно трёх):

Уязвимость в rpcbind (CVE‑2017‑8779)

Описание: уязвимость типа «утечка памяти» (memory leak) в обработке строк XDR. Злоумышленник может отправить специально сформированные пакеты, вызывающие непрерывное выделение памяти без её освобождения, что приводит к отказу в обслуживании (DoS).

Тип атаки: DoS (Denial of Service).

Уязвимости в NFS (версии до 3.x)

Описание: устаревшие версии NFS часто содержат слабые настройки безопасности. Например, отсутствие аутентификации или разрешения на монтирование без ограничений позволяют неавторизованным пользователям получить доступ к файловым системам. Это может привести к чтению, изменению или удалению данных.

Тип атаки: несанкционированный доступ к файлам, эскалация привилегий.

Уязвимость в RPC‑сервисах (CVE‑2011‑2753)

Описание: уязвимость в реализации RPC‑протокола, позволяющая удалённо выполнить произвольный код или вызвать сбой службы. Особенно актуальна для старых версий rpcbind и связанных RPC‑сервисов.

Тип атаки: удалённое выполнение кода (RCE) или DoS.

Задание 2

Чем отличаются эти режимы с точки зрения сетевого трафика?

1. SYN (Stealth Scan)
Использует только флаг SYN. Создает «полуоткрытое» соединение. Самый надежный способ определить открытые порты. Генерирует стандартный TCP-трафик (SYN → SYN-ACK/RST).
2. FIN Scan
Отправляет пакет с флагом FIN. Открытые порты игнорируют такой пакет (нет ответа), а закрытые отвечают RST. Создает аномальный TCP-трафик, который может не фиксироваться старыми системами логирования.
3. Xmas Scan
Пакет с флагами FIN, PSH, URG (как украшенная елка). Поведение аналогично FIN, но выглядит более подозрительно в логах.
4. UDP Scan
Отправляет UDP-пакеты без рукопожатия. Не показывает состояние «слушает», только «закрыт» или «не отвечает». Генерирует ICMP-ошибки при закрытых портах.

Как отвечает сервер?

1. TCP: Открытые порты — SYN-ACK; Закрытые порты — RST.
2. UDP: Открытые порты — нет ответа; Закрытые порты — ICMP Destination Unreachable.
3. Фильтрация: Если фаервол блокирует пакеты или ICMP, сканер видит статус filtered.
