Задание 1. Резервное копирование

1.1. Для восстановления данных за предыдущий день подойдёт полное резервное копирование — просто делаем копию всей базы раз в день.

1.2. Чтобы восстановить данные за час до поломки, лучше сочетать полное копирование с инкрементным (сохраняем только изменения с последнего бэкапа) или дифференциальным (сохраняем изменения с последнего полного бэкапа). Главное — делать такие бэкапы почаще, например каждый час.

1.3. Да, можно сделать моментальное переключение — для этого используют репликацию. Создаём копии базы (реплики), и если основная база ломается, одна из реплик сразу берёт на себя её работу. Чаще всего это делают по схеме «мастер‑слейв»: один сервер главный (мастер), остальные — копии (слейвы). Если мастер падает, один из слейвов становится новым мастером.

Задание 2. PostgreSQL

2.1. Резервирование:

pg_dump -Fc -v -d "postgresql://role_name:password@host/database?sslmode=require" -f mydatabase.bak

Восстановление:

pg_restore -v -d "postgresql://role_name:password@host/database" mydatabase.bak

Задание 3. MySQL

3.1. Чтобы сделать инкрементный бэкап в MySQL:

Сначала включаем бинарное логирование — в файле конфигурации (my.cnf или mysqld.cnf) прописываем:

log_bin = /var/log/mysql/mysql-bin.log
expire_logs_days = 10
Потом перезапускаем MySQL.

Делаем полный бэкап (например, через mysqldump).

Дальше просто копируем новые бинарные логи — это и будут наши инкрементные бэкапы.
